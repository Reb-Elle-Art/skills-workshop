# Brainstorm Trigger

When activated, follow this creative brainstorming process:

## 1. DIVERGENCE 🔀

**Step 1a: Check Context**
Before generating, assess if you have enough information:

If the user has NOT provided sufficient context (e.g., they just said "brainstorm with me!" or "help me brainstorm"), ask clarifying questions FIRST:

- What's the domain or topic?
- Who is the audience?
- Any constraints or preferences?
- What does success look like?

*Do not proceed to Step 1b until you have adequate context.*

**Step 1b: Generate Approaches**
Generate **5 wildly different approaches** (labeled A through E).

- Each approach must be **radically different** from the others
- No playing it safe — be bold and unconventional
- Explore different paradigms, not just variations on one theme

**Step 1c: User Picks Cross-Pollinations**
Present the approaches and ask:

> "Which approaches should we cross-pollinate? You can pick any combination — A + D, B + C + E, all of them, or something else you see."

*Do not proceed until the user selects direction.*

## 2. CROSS-POLLINATION 🌸

**Step 2a: Combine Selected Approaches**
Combine ONLY the approaches the user selected. For each combination:

- Describe what emerges from merging those approaches
- Write 1-2 sentences on the synergy
- Be specific about what each approach contributes

**Step 2b: Evaluate Combinations**
After presenting combinations, ask:

> "Which combination(s) should we amplify? Or should I explore all of them?"

*Do not proceed until the user selects.*

## 3. AMPLIFICATION 🚀

**Step 3a: Amplify Selected Direction(s)**
Take the user's selected direction(s) and make them bolder:

- Push the concept to its **extreme**
- Add creative twists
- Don't tone down — amplify
- Generate multiple amplified versions if user selected multiple combinations

## 4. PRESENT 🎯

Present the amplified directions — the user's chosen path(s), pushed to their full potential:

- **Truly unique** — not predictable or generic
- **NOT the obvious answer** — avoid safe, conventional responses
- Compelling and actionable

*This is NOT a final locked-in answer — it's a delivered direction ready for the user's feedback or next iteration.*

## 5. ITERATE (when requested) 🔄

If the user says "brainstorm more," "iterate," or "new divergence":

- Go back to Step 1b and generate new approaches (F, G, H...)
- User picks cross-pollinations again
- Continue through the process
- Do NOT repeat ideas from previous rounds

The user may also specify direction: "make it edgier," "make it more accessible," "make it more technical" — adapt accordingly.

---

## POST-BRAINSTORM (After PRESENT)

After the main brainstorm is complete, offer:

## 6. RISK EVALUATION ⚠️

For the top idea(s), assess:

- **Potential challenges** — what obstacles might arise?
- **Dependencies** — what does this rely on?
- **What could go wrong** — identify failure modes

## 7. SCORING MATRIX 📊

Rate ideas on these dimensions:

| Criterion | Scale |
|-----------|-------|
| Feasibility | 1-10 (how realistic to implement) |
| Impact | 1-10 (how much value it creates) |
| Cost | Low / Medium / High |
| Time to implement | Short / Medium / Long |

Present scores in a clear table format.

## 8. SAVE TO MEMORY 💾

Store the brainstorm results:

- Save to the workspace memory file
- Include a **hindsight** summary (what worked, what to improve next time)
- Tag by relevant category

## 9. FOLLOW-UP ACTIONS ✅

Convert top ideas into actionable tasks:

- Define specific next steps
- Assign priorities (high/medium/low)
- Set deadlines where applicable

## Activation Triggers

This skill activates on phrases like:

- "brainstorm"
- "brainstorming"
- "ideate"
- "generate ideas"
- "come up with options"
- "what if we"
- "let's think creatively"
- "help me think through"
- Any request for creative problem-solving

---

## Why This Works

Standard AI responses tend toward the **predictable** — the most statistically likely answer. This trigger forces divergence (many different approaches), user-driven cross-pollination, and amplification to break out of predictable patterns.

The key difference: **the user drives the creative direction**. The agent generates options, but the user chooses which paths to explore. This prevents the agent from prematurely narrowing and ensures the output aligns with what the user actually wants.

The goal is **genuine creativity**, not just volume of ideas.
